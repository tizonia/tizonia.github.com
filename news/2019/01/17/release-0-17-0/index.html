Spotify login issues fixed in Tizonia 0.17.0

Jan 17, 2019 • juanrubio

Tizonia version 0.17.0 codenamed “Nijar” is out with 23 commits, 995 files changed, 1552 insertions(+), 1154 deletions(-).

This release primarily fixes a very important issue that was reported soon after 0.16.0 was released back in December. Users noticed that they could not access Spotify using Tizonia:

Spotify: “unable to login offline: no such user” #531

This problem turned out to be a regression introduced in v0.16.0. I would like to thank all the users that contributed with feedback in #531. This feedback ended up being crucial in finding the actual root cause of the failed login attempts. Hats off to all of you!

Binaries and distribution

As usual CHANGELOG.md contains all the details.

Tizonia v0.17.0 will be soon in the following distribution channels:

  • Bintray (Debian packages)
  • AUR (Arch Linux)
  • Snap Store.
  • docker-tizonia.

Credits

In addition to the users of #531, I would also like to thank the following people that contributed with PRs during this cycle:
